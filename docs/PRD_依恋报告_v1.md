## 一、产品全貌

### 1.1 核心交付物

一份16页PDF报告，包含：

1. 用户的依恋类型（四象限定位：安全型/焦虑型/回避型/恐惧型）

5. 本周可以开始做的3个具体行为练习（可操作，不是口号）
6. 附录：你的焦虑维度分 / 回避维度分的位置说明

## 二、问卷设计（ECR-R简化版）

### 2.1 量表逻辑

使用Experiences in Close Relationships-Revised（ECR-R）的12题缩减版，包含：
- **焦虑维度**（Anxiety）：6题，测量"担心被抛弃、需要确认感情、情绪波动"
- **回避维度**（Avoidance）：6题，测量"情感疏离、不舒适于亲密、压抑情感需求"

每题7分制Likert量表（1=完全不符合，7=完全符合）。

### 2.2 12道题目（Tally表单内容）

**说明文字（表单顶部）：**
> 以下题目描述你在亲密关系中的感受和行为方式。
> 请基于你在感情关系中的真实体验作答，不存在对错之分。

**焦虑维度（6题，A1-A6）：**

| 编号 | 题目 |
|------|------|
| A1 | 我很担心自己会被另一半抛弃 |
| A2 | 我需要对方经常表达对我的感情，才能感到安心 |
| A3 | 如果对方没有及时回消息，我会开始胡思乱想 |
| A4 | 当对方显得疏远时，我会变得焦虑或愤怒 |
| A5 | 我比大多数人更担心感情不够稳固 |
| A6 | 我总是觉得自己对感情投入得比对方多 |

**回避维度（6题，B1-B6）：**

| 编号 | 题目 |
|------|------|
| B1 | 我不太舒适于向伴侣倾诉内心深处的想法和感受 |
| B2 | 在关系中过于亲密会让我感到不自在 |
| B3 | 我倾向于不依赖任何人，靠自己解决问题 |
| B4 | 需要别人会让我感到有点不舒服 |
| B5 | 当伴侣想要在情感上更亲近时，我会有些退缩 |
| B6 | 我在感情中不太擅长表达脆弱和需求 |

### 2.3 分型算法

```python
def classify(answers: dict) -> dict:
    """
    answers: {
        'A1': int(1-7), 'A2': int(1-7), ... 'A6': int(1-7),
        'B1': int(1-7), ... 'B6': int(1-7)
    }
    """
    anxiety_score = sum(answers[f'A{i}'] for i in range(1, 7)) / 6
    avoidance_score = sum(answers[f'B{i}'] for i in range(1, 7)) / 6
    
    # 分型阈值（基于ECR-R标准化数据，中位数约为3.5）
    THRESHOLD = 3.5
    
    if anxiety_score < THRESHOLD and avoidance_score < THRESHOLD:
        type_code = "SECURE"      # 安全型
    elif anxiety_score >= THRESHOLD and avoidance_score < THRESHOLD:
        type_code = "ANXIOUS"     # 焦虑型
    elif anxiety_score < THRESHOLD and avoidance_score >= THRESHOLD:
        type_code = "AVOIDANT"    # 回避型
    else:
        type_code = "FEARFUL"     # 恐惧型（高焦虑+高回避）
    
    return {
        "type": type_code,
        "anxiety_score": round(anxiety_score, 2),
        "avoidance_score": round(avoidance_score, 2)
    }
```

## 三、报告内容规格

### 3.1 四类型报告内容库（静态预生成）

每种类型报告的核心内容**预先写好**（非每次调用LLM生成），降低成本至趋近于零，同时保证质量稳定性。

PDF由**固定内容模块 + 动态数据插值**组成：
- 固定内容：类型详解、行为模式、冲突解析、练习方案（按类型选库）
- 动态插值：用户昵称、焦虑分、回避分、分数在坐标轴上的位置

### 3.2 页面结构（16页）

| 页码 | 内容 | 字数/内容量 |
|------|------|------------|
| 封面 | 类型名称 + 用户昵称 + 知我实验室Logo | — |
| P2 | 你的依恋坐标图（焦虑轴×回避轴，标注你的位置） | 图表+200字说明 |
| P3-4 | 你是谁：类型深度解读（心理机制来源，原生家庭角度） | 600字 |
| P5-7 | 你在关系中的5个典型行为模式（每个模式含具体场景举例） | 每个150字×5 |
| P8-9 | 你和哪类人最容易起冲突，为什么（含互动机制图） | 400字+图 |
| P10-11 | 你和哪类人相处最舒服，怎么识别 | 300字 |
| P12-14 | 本周开始做的3个具体行为练习（含操作步骤，不是口号） | 每个200字×3 |
| P15 | 你的数据详情（焦虑分/回避分/各题得分分布） | 图表 |
| P16 | 尾页（知我实验室介绍 + 引流钩子） | — |

### 3.3 四类型内容要点（写报告时的核心差异化要求）

**焦虑型（高焦虑/低回避）：**
- 行为模式重点：反复确认行为、分手威胁、情绪化表达、粘人-愤怒循环
- 核心心理机制：对被抛弃的高度警觉，源于早期不一致的养育反应
- 练习方向：容忍不确定性训练、情绪调节窗口、降低确认频率的具体步骤

**回避型（低焦虑/高回避）：**
- 行为模式重点：情感关闭、主动制造距离、对"控制感"的防御需求、以忙碌作为情感防护
- 核心心理机制：情感表达被惩罚的学习经历，用独立防止受伤
- 练习方向：小剂量脆弱暴露练习、"停留在不适感中"训练、用语言替代行为退缩

**焦虑型（即恐惧型，高焦虑/高回避）：**
- 最复杂，同时渴望亲密又害怕被伤害
- 行为模式：推拉行为、自我破坏关系的无意识行为、关系里的自我价值感极低
- 练习方向：区分当下威胁与童年威胁、建立自我安全感基础练习

**安全型（低焦虑/低回避）：**
- 不作为"完美型"输出，而是描述"你在哪些方面比较稳定，在什么情况下会被激活"
- 给出维持安全感的具体方式，以及识别不安全伴侣时的早期信号

## 四、技术栈

### 4.1 最小可行技术方案

```
问卷：        Tally（免费，支持条件逻辑，Webhook输出）
后端：        Python脚本（本地 or Railway/Render免费实例）
分型计算：    纯Python，无需LLM
PDF生成：     ReportLab（Python库）or WeasyPrint（HTML→PDF）
文件托管：    阿里云OSS / Cloudflare R2（按需付费，几乎免费）
自动发链接：  
  - 闲鱼端：闲鱼虚拟商品"付款后自动发"功能，直接填OSS链接
  - 小红书端：微信机器人（itchat/企业微信）自动回复含PDF链接
```

### 4.2 完整数据流

```
Tally提交 → Webhook触发Python脚本
    → 读取answers
    → 调用classify()得到type_code + scores
    → 从内容库读取对应类型的报告模板
    → 插入用户数据（昵称、分数、坐标图数据）
    → 生成PDF → 上传OSS → 返回带时效的下载链接（7天有效）
    → 调用微信机器人API / 触发邮件发送
```

### 4.3 PDF生成方案对比

| 方案 | 优点 | 缺点 | 推荐场景 |
|------|------|------|---------|
| ReportLab | 纯Python，完全可控，快速 | 排版写起来繁琐，代码量大 | 数据类页面（坐标图、分数表）|
| WeasyPrint（HTML→PDF） | HTML/CSS写报告，排版灵活 | 图形库支持弱，字体中文需配置 | 文字内容页 |
| **推荐：混合方案** | WeasyPrint做正文，ReportLab做图表，合并 | 复杂度略高 | 正式版 |
| **MVP方案：纯WeasyPrint** | 快速上线 | 图表简单 | 前30天先跑通 |

## 八、风险防控

### 8.1 分型不准确

**触发场景：** 用户读完报告说"和我完全不像"  
**根因：** 12题量表在边界情况（分数接近3.5）分型不稳定  
**处理方案：**
- 报告P2坐标图展示用户的精确位置，如果位置接近中心，文字注明"你的依恋风格较为灵活，以下描述选取对你影响较大的那一侧"
- 报告结尾加一段："依恋风格会随情境和关系伴侣不同而变化，这份报告反映的是你的主导模式"

### 8.2 恶意退款

**触发场景：** 截图PDF后申请"商品与描述不符"退款  
**防控：**
- 闲鱼商品详情放示例报告封面+第3页（展示内容样式，付款者知情）
- 报告PDF加用户昵称+生成日期水印（低成本证明是定制化交付）

### 8.3 PDF发送失败

**触发场景：** OSS链接生成但微信机器人发送失败  
**降级方案：** 同时发送邮件（如果用户填了邮箱）；设置定时重试（30分钟内最多3次）；失败后记录日志，每日检查一次手动补发
